本项目致力于使用深度强化学习（Deep Reinforcement Learning）实现双轮差速无人车（UGV）的自主导航控制，并面向真实的物理样机进行 Sim2Real（仿真到现实）部署准备。

本项目基于 NVIDIA 官方开源的下一代物理仿真平台 Isaac Lab，利用 GPU 张量并行技术实现数千个环境的同步推演。相比传统仿真，本项目直接在物理引擎的张量 API 上进行端到端控制，涵盖了差速动力学、地面摩擦力建模以及传感器延迟模拟。

### 3. 强化学习建模 (RL Modeling)
针对物理部署的需求，本项目对 RL 三要素进行了针对性设计：

#### A. 状态空间设计 (Observation Space)
为了实现平移不变性与感知动态特性，采用 **3 帧堆叠 (Frame Stacking)** 方案：
- **单帧特征 (7 维)**：包含相对目标点的距离 $d$、目标点相对航向的 $\sin(\alpha)/\cos(\alpha)$、左右轮实时转速、以及上一时刻动作反馈。
- **最终输入 (21 维)**：将过去 3 帧拼接，使 MLP 网络能隐式感知加速度与传感器滞后。
- **在线归一化**：通过 RunningStandardScaler 实现输入特征的实时标准化。

#### B. 动作空间与平滑处理 (Action Space)
- **输出定义**：2 维连续值，映射至左右轮的目标角速度。
- **动作平滑滤波**：引入低通滤波机制 $Action_{real} = \tau \cdot a_t + (1-\tau) \cdot a_{t-1}$，其中 $\tau=0.2$。此设计可滤除高频抖动，防止真机部署时损坏减速齿轮箱。

#### C. 奖励函数设计 (Reward Shaping)
所有奖励组件单步截断在 $[-1, 1]$，确保梯度稳定性。
1. **步数惩罚**：-0.002，鼓励最短路径。
2. **接近奖励 (Approach Reward)**：基于势能差设计。$R = 1.0 \cdot (d_{t-1} - d_t)$，只有缩短目标距离时才给正奖。
3. **动作平滑惩罚**：惩罚动作变化的平方和，引导网络生成平滑曲线。
4. **目标达成奖励**：每到达一个点给予 $+0.5$ 分；彻底完成 5 个点给予 $+1.0$ 分大奖。
5. **目标切换修正**：在切换目标点的瞬间刷新距离记忆，消除跨目标点带来的巨大瞬时负奖励。

### 4. 算法与训练细节
#### 核心算法
- **PPO (Proximal Policy Optimization)**：利用 SKRL 实现。采用逻辑与物理分离架构，通过 64 个并行环境在 GPU 上直接收集 Transition 数据。

#### 网络架构
- Actor/Critic：独立的 MLP 结构 `[256, 128, 64]`。
- 激活函数：ELU，提供比 ReLU 更平滑的梯度流。
- 初始化：采用 Orthogonal 正交初始化，gain 设为 $\sqrt{2}$。

### 3. 强化学习建模 (RL Modeling)
针对动态避障的复杂性，本项目对状态感知和安全性进行了设计：

#### A. 状态空间设计 (Observation Space)
为了捕捉障碍物的相对速度，采用 **3 帧堆叠 (Frame Stacking)** 结合数据降维：
- 雷达池化 (Avg-Pooling)：将 360 根雷达射线按每 10 度进行最小距离提取，降维至 36 维，保留最危险的避障特征。
- 单帧特征 (41 维)：包含目标距离、目标相对方向 ($\sin/\cos$)、实时轮速反馈及 36 维池化雷达数据。
- 最终输入 (123 维)：堆叠 3 帧数据，使纯 MLP 网络能够通过距离变化隐式推算动态障碍物的速度矢量。

#### B. 动作空间与平滑处理 (Action Space)
- 输出定义：2 维连续值 $[-1,1]$，映射至左右轮的目标角速度（Max: $15\text{rad/s}$）。
- EMA 动作滤波：引入指数移动平均滤波 $Action_{real} = 0.2 \cdot a_t + 0.8 \cdot a_{t-1}$。该机制在软件层面模拟了电机的惯性，强制物理输出平滑化，消除算法探索期的震荡。

#### C. 奖励函数设计 (Reward Shaping)
采用稠密奖励与高斯斥力场相结合，单步奖励截断在 $[-0.5,0.5]$：
- 步数惩罚：-0.005，强力驱使小车拒绝原地发呆。
- 接近奖励：$R_{app} = 2.5 \cdot (d_{t-1} - d_t)$，作为主要的全局引导拉力。
- 高斯靠近惩罚：基于雷达最近距离 $d_{min}$ 设计斥力场：$R_{prox} = -0.5 \cdot \exp\left(-\frac{d_{min}^2}{2\sigma^2}\right)$。当小车靠近障碍物时，惩罚随距离呈指数级剧增。
- 碰撞 / 越界极刑：大额固定惩罚 $-1.0$，直接触发回合终止。
- 任务完成大奖：固定奖励 $+1.0$，引导模型完成最终的精准触达。

### 4. 算法与训练细节
#### 核心算法
- SB3 PPO：采用稳定版 PPO 算法，结合在线学习率衰减（Linear Schedule）和梯度裁剪（Max Grad Norm: 0.5）。

#### 网络架构
- Actor/Critic：共用或独立的 MLP 结构 `[512, 256]`。
- 激活函数：ELU，确保在处理雷达点云数据时的梯度响应更加细腻。

### 3. 强化学习建模 (RL Modeling)
针对变摩擦力环境与实车部署需求，本项目进行了如下建模设计：

#### A. 状态空间设计 (Observation Space)
采用 **3 帧堆叠 (Frame Stacking)** 捕捉动力学特征：
- 感知特征 (41 维)：包含当前轮速反馈 (2D)、目标点局部距离 (1D)、目标相对航向角 $\sin/\cos$ (2D) 及 36 维池化雷达数据（从 Bpearl 模式降采样）。
- 时序输入 (123 维)：堆叠 3 帧观测，使网络能够感知冰雪路面打滑引起的速度波动及减速带带来的垂直振动。

#### B. Sim2Real 退化管道 (Degradation Pipeline)
为了确保仿真策略在 Jetson Nano 硬件上的可迁移性，在动作下发前引入物理退化模拟：
- 动作延迟：模拟约 80ms (2 帧) 的指令传输与系统调度延迟。
- 电机死区：设定 0.05 的动作死区，模拟真实电机在低占空比下无法克服静摩擦力的现象。
- EMA 平滑：引入 $\alpha=0.5$ 的动作指数移动平均滤波，强制物理输出平滑化，保护实车电机驱动器。

#### C. 奖励函数设计 (Reward Shaping)
采用里程碑驱动的混合奖励机制：
- 时间惩罚：-0.02/step，逼迫智能体尽快完成泊车，减少无效原地摆烂行为。
- 势能引导：R_app = 15.0 × (d_{t-1} - d_t)，提供持续的前进动力。
- 地形里程碑：小车完全越过地毯区及进入泊车区时，分别发放固定的阶跃奖励 (+2.0)，解决稀疏奖励下的探索难题。
- 对齐奖励：仅在泊车区激活。基于 Cos(Yaw_curr - Yaw_goal) 计算，引导小车在最后阶段调整车头角度。
- 极刑判定：撞击赛道侧墙或越界处以 $-50.0$ 惩罚并立即重置。

### 4. 算法与训练细节
#### 核心算法
- 非对称 PPO：利用 AsymmetricMlpExtractor 架构。Actor 仅接收 123 维传感器观测（凡人视角），Critic 在训练阶段可接收包含特权信息的完整特征，提升值函数评估的准确性。

#### 物理参数校准
- 驱动模式：采用速度控制（Velocity Control），将关节阻尼（Damping）设定为 $10$，确保符合实车。
- 碰撞几何：减速带高度优化为 $5\text{mm}$，匹配 Jetbot 微型底盘的通过性，避免发生硬性碰撞锁死。

#### 网络架构
- 规模：Policy 采用 `[512, 256, 128]` 结构。
- 激活函数：ELU，在动作空间边界处提供比 ReLU 更平滑的梯度衔接。

## ➡️ Task 4: 无信控十字路口多机协同通过
本任务要求 4 辆双轮差速无人车（Jetbot）在包含低摩擦力中心区的十字路口场景中，通过互相感知与策略博弈，在不发生物理碰撞的前提下安全、高效地通过。

### 1. 任务描述
* **初始状态**：4 辆无人车分别出生在十字路口（北、南、东、西）四个端点的 0.8m 窄道内。
* **环境配置**：
  - 赛道布局：总跨度 20m 的“十”字型走廊。每个分支通道宽度仅为 0.8m（约两倍车宽），中心 0.8x0.8m 区域为交叉汇合点。
  - 路面材质：四个分支通道为柏油路面（高摩擦 0.8），中心交叉区域为冰雪路面（极低摩擦 0.1），增加过弯与避让时的动力学不确定性。
  - 异构性能：为模拟现实差异，每辆车具备不同的最大线速度（如：1.0m/s、0.8m/s、1.2m/s、0.6m/s），强制模型处理非对称的时间差。
* **任务目标**：4 辆车随机分配目标点（不以自身起点为终点），协同通过中心路口并到达目的地。
* **判定标准**：
  - 个体成功：单车质心与目标点距离 $< 0.3\text{m}$，随后系统接管并锁定位置。
  - 团队胜利：所有车辆全部安全抵达目标点。
* **挑战**：由于车道狭窄，对向或侧向车辆在中心点极易发生物理死锁。模型必须学会“提前减速”、“主动让行”等高级协同行为。

### 2. 代码架构与运行指南

### 3. 强化学习建模 (RL Modeling)
针对多智能体博弈，本项目基于 MAPPO 算法进行了针对性设计：

#### A. 状态空间设计 (Observation & State)
采用分层感知架构以支持 CTDE：
- 局部观测 (Actor - 150 维)：基于 3 帧堆叠。单帧包含自身轮速、目标相对位姿、其余 3 辆队友的相对距离与方向、以及 36 维池化雷达数据。
- 全局状态 (Critic - 28 维)：仅在训练阶段使用的 “上帝视角”。包含所有 4 辆车的绝对坐标 $(X,Y)$、绝对航向角、线速度矢量及各自的目标位置。

#### B. 动作空间与物理封锁 (Action Space)
- 输出定义：2 维连续值 $[v,\omega]$。
- 物理约束：采用线性映射将线速度 $v$ 强制限定在 $[0,MaxSpeed]$，从物理层封锁倒车权限，强制小车仅能通过前行或停止来解决死锁。
- 平滑机制：引入 $\alpha=0.5$ 的动作 EMA 滤波，模拟电机惯性并抑制多机干扰下的策略震荡。

#### C. 奖励函数设计 (Reward Shaping)
采用共享惩罚与个体激励相结合的机制，单步截断在 $[-1.0,1.0]$：
- 势能引导：$R_{app}=2.0\cdot(d_{t-1}-d_t)$，利用距离差提供单车导航拉力。
- 高斯共享斥力：当任意两车距离 $<0.8\text{m}$ 时，触发 $R_{prox}=-0.5\cdot\exp\left(-\frac{d^2}{2\sigma^2}\right)$。惩罚由相关的两辆车平分，迫使双方共同承担避让义务。
- 动作突变惩罚：针对 $[v,\omega]$ 的平方差项，单步权重 $-0.01$，确保路径平滑。
- 里程碑奖励：成功越过中心十字路口（里程碑）给予 $+2.0$。
- 连坐机制 (Capital Punishment)：若任意一辆车撞墙或发生车间碰撞，全队统一扣除 $-20.0$ 并立即重置回合。
- 团队完赛奖：全队完成任务后，根据剩余时间步发放巨额共享奖励（Base: 50.0），鼓励群体效率。

### 4. 算法与训练细节
#### 核心算法
- Custom MAPPO：基于纯 PyTorch 实现，针对 Isaac Lab 的并行张量流进行了优化。
- 集中式评判：Critic 网络评估全局状态以减少多智能体环境中的非平稳性（Non-stationarity）问题。

#### 稳定性控制
- KL 自适应学习率：实时监测策略更新前后的 KL 散度，若波动超过 $0.008$ 则自动缩减学习率，防止策略崩溃。
- 控制频率：RL 控制频率设定为 50Hz（Decimation: 2），在保证物理精确度的同时提升样本利用率。
- 参数初始化：全连接层采用正交初始化（Orthogonal Initialization），输出层增益设定为 $0.01$，确保训练初期动作温和。
